## GRADIENT MODE REFINEMENT
- Make all bins the same size - whatever Brush bins size is make patterns, gradients, all bins same size
- Gradient colors should not be spread across the entire canvas
- The gradients are currently acting like a wallpaper, where regardless of tool, it's 
  always showing the same thing. It's acting more like a mask that tools reveal the gradient
  from. That's not how it should work.
- Gradient colors should stretch/fit inside / spread across the drawing tool boundaries
  - When drawing with a line, from the start of the line to the end of the line
  - When drawing with a brush, from the start of the brush stroke to when user lifts it
  - When using a fill, should fill with gradient at the boundaries of the fill
  - When using a shape, should fill with gradient at the boundaries of the shape
  - Rect, Poly line, Ellipse - all should just use the gradient colors to plot them 
    - As the stroke/at the extents and flow around the shape
  - Filled Rect, Filled Poly line, Filled Ellipse - all should fill like fill tool
    but only within the shape of the shape, the boundaries, etc.
  - When filling selections should fill to the extents of the actual selection
- Every time the user draws something with a gradient it should determine:
  - Where on X to start the gradient from
  - Where on Y to start the gradient from
  - Where on X to end the gradient at
  - Where on Y to end the gradient at
  - Including filled areas with scanline algorithm, etc for shapes that are not simple.
- The gradient editor should provide a dialog that looks like the image
  - PLANS/GRADIENT-EDITOR-DIALOG.png
  - This editor dialog should be entirely themeable
    - Following suit and infering what should be done and how things are named
  - Use this URL to understand how gradient editing should work
    - #fetch https://www.teachucomp.com/how-to-use-the-gradient-editor-in-photoshop-instructions/

  - [x] PATTERN TILE MODE (<= 512px) ... SHIFT+TAB
    - Only available if canvas size is <= 512 x 512
    - If canvas is > 512px in any dimension - show as grayed out/disabled
    - Checkable option - OFF by default
    - Checked state IS SAVED with file for later edits

### PATTERN TILE MODE FUNCTION
- ON: 
  - Canvas content repeats one time in every direction and diagonals: 

    ```
    - Where [C] is cloned copy of [S]OURCE 
    - [C] regions are not part of the actual canvas in the file, etc.
    - They are only shown and active when View -> PATTERN TILE MODE is active.
    - The are rendered and laid out in a 3x3 grid to assist in the creation of patterns and tiles.

    3x3 grid:

    C C C
    C S C
    C C C

    S = SOURCE canvas (actually part of the file when saved, etc.)
    C = CLONED canvas
    ```

  - All regions and clones are editable and apply edits to the [S]
  - When [S] changes, update in real time [C]
  - Show cursors, assistants, etc. in [C] just as you do [S] but:
    - Do not have duplicate cursors or assistants.
    - For example if the user is painting on a [C], show the same cursors/UX
      as when they are painting on [S], allow same functions.
    - If the user is using color picker on [C], show same cursor/UX, etc.
  - SELECTIONS should not be rendered more than once. 

- OFF:
  - Normal mode as it is now.
